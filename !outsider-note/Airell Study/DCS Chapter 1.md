[[DCS UNIMEL]]

==Data - ===
	Fakta mentah atau maklumat yang belum diproses

==Komputer -==
	-Peranti yang boleh memproses data
	-Peranti elektronik yang boleh melakukan pengiraan dan keputusan lebih laju dari manusia
	-Hanya boleh memahami machine language (1 0, binary code)

==Jenis bahasa programming -==
	Machine Language
	Assembly Language
	High level language (HLL)

==C++ relationships-==
	Program-
		Assembler, compiler dan interpreter digunakan untuk menterjemah bahasa assembly language dan HLL kepada Machine Code
	Compiler-
		Menterjemah arahan dari HLL sekali gus
	Interpreter -
		Menterjemah arahan HLL satu2
	Assembler -
		Menukar assembly language kepada machine code

==Comments -==
	Digunakan untuk dokumentasi tetapi tidak wajib dalam program, ditulis mengikut kehendak programmer
		Contoh:
		// untuk satu baris
		/* untuk banyak baris

==Pre-processor Directive-==
	-dikenali sebagai header file
	-bukan kod yang boleh dilaksanakan tetapi arahan kepada compiler
	-bermula dengan #
	-Contoh: \#include < iostream > iaitu input dan output seperti papar pada skrin
==Function==
	-Semua c++ program mempunyai function, tetapi satu wajib dinamakan *main*
	-Function adalah satu blok code yang menjalankan tugas spesifik
	-bermula dengan void atau int, diikuti dengan ()

==Braces==
	- digunakan untuk menandakan permulaan dan penutupan blok kod dalam program {}
	- apa2 ditulis selepas } diabaikan oleh compiler

==Statement==
	- Function terdiri daripada urutan statement yang melakukan kerja function
	- diakhiri dengan semicolon ;

==Program Development Life Cycle(PDLC)==- Analysis (menyemak spesifikasi program)
	- Design (Pseudocode and Flowchart)
	- Coding (Menukarkan design menjadi program menggunakan bahasa programming)
	- Testing/debugging (Memastikan program betul dan tiada ralat)
	- Maintenance (mengemaskini dengan penambahan atau pembetulan agar program tahan lama)
